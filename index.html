Cours GIT

Qu'est-ce que GIT

Git est un outil de versionning et permet d'avoir une traçabilité et un historique de tous les fichiers du projet.

il permet également de travailler en equipe : envoyer nos modification aux collegues et recuperer les modifications des collegues.

Dans la traçabilité des fichiers, GIT nous donne toujours 3 infos: Qui, Quand et Pourquoi ?

url: https://git-csm.com

Les commandes de base



initialiser un projet
git init

voir le statut des fichiers
git statut

ajouter des fichiers
git add'fichier'

ajouter TOUT LES FICHIERS:
git add.

commit(enregistrer) les fichiers ajoutés
git commit
un commit necessite toujours un nmessage

commit avec message:
git commit -m "message"

commit avec add+message:
git commit -a -m "message"

commit avec add + message (fonctionne sur les fichiers deja add précédemment):
git commit -a -m "message"

Correction des fichiers

voir les differents commit:
git log

corriger les modification sur un fichier (pas encore add):
git checkout 'fichier'

supprime un add:
git rest'fichier'

git rest.

supprimer le dernier commit:
git rest HEAD^

Supprimer les deux derniers commit:
git rest HEAD^^

La notion de brache

une branche est une copie paralelle du projet. elle permet de devolopper en paraleLle de nouvelles fonCtionnalités, lorsqu'on considere des nouveautés operationnelles, on peut les rappatrier sur la version principale qu'on appelle la branche "Master".

Voir les branches:
git branch

Creer une nouvelle branche:
git branch'nom'

Se placer sur une branche:
git checkout 'nom'

Voir les différences entre deux branches:
git diff 'nom'..master

APPORTER les modifications effctués sur une brache dans MASTER (il faut etre sur master)
git checkout master
git merge.'nom'

REMPLACER master par une branche:
git checkout master
git fetch.'nom'
Le fetch va literalLement remplacer master par la braNche. Il s'agit donc d'une grosse modification dans le projet (ex: version 2)
